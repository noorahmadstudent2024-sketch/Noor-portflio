# Playwright Testing Skill

A skill that performs visual testing of your website using Playwright.

## Purpose

This skill uses Playwright to conduct visual testing of your website by taking screenshots, checking responsive design across different viewport sizes, and verifying UI elements follow best practices.

## Knowledge Base

The skill incorporates knowledge of:

- Common viewport sizes for responsive testing
- Visual regression testing methodologies
- UI element verification techniques
- Cross-browser compatibility checking
- Mobile-first responsive design principles
- Visual consistency standards
- Screenshot comparison techniques
- Visual testing best practices

## Common Viewport Sizes

- Mobile: 375x667 (iPhone SE), 414x896 (iPhone 12 Pro Max)
- Tablet: 768x1024 (iPad), 834x1194 (iPad Pro)
- Desktop: 1280x800, 1366x768, 1920x1080
- Large Desktop: 2560x1440, 3840x2160

## What to Look For

- Visual consistency across browsers
- Proper layout and spacing
- Correct typography rendering
- Image and media display
- Form element appearance
- Button and interactive element states
- Navigation behavior
- Responsive breakpoints
- Scrolling behavior
- Loading states

## Capabilities

- Take screenshots of entire pages or specific elements
- Test responsive design across multiple viewport sizes
- Verify UI elements are properly displayed
- Check for visual regressions
- Capture before/after comparisons
- Test different browser contexts
- Validate layout consistency
- Verify component rendering
- Check for visual anomalies
- Generate visual test reports

## Usage Examples

- "Test my homepage responsiveness"
- "Take screenshots of my projects page in different sizes"
- "Verify my navigation looks correct on mobile"
- "Check how my contact form appears on tablet"
- "Test visual consistency across browsers"
- "Capture my site on common viewport sizes"
